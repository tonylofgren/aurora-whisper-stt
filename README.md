# Aurora Whisper

> *Generated by [aurora@aurora-smart-home (ha-integration-dev skill)](https://github.com/tonylofgren/aurora-smart-home)*

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-orange.svg)](https://github.com/hacs/integration)

A Home Assistant integration that adds a **Speech-to-Text (STT) engine** built around an
**OpenAI-compatible** Whisper REST API. The default is [staik.se](https://staik.se/) with
KB-Whisper (tuned for Swedish), but `base_url` is configurable so it also works against OpenAI, a
local Whisper server, or any `/v1/audio/transcriptions`-compatible endpoint.

> **About staik.se (the default endpoint):** an OpenAI-compatible API running on dedicated GPU
> hardware in Sweden. Your prompts and responses stay in Sweden — no third-party sharing, no
> training on your data, GDPR-compliant.

## What this does

The integration registers an `stt` entity you can select as the STT engine in an Assist pipeline
(**Settings → Voice assistants**). When you speak, Home Assistant streams raw PCM audio to the
entity, which wraps it as WAV and POSTs it to `POST {base_url}/audio/transcriptions`
(multipart/form-data, Bearer auth). The transcribed text is fed back into the pipeline.

- All HTTP is async via Home Assistant's shared `aiohttp` session — no blocking calls.
- The entity advertises the languages `sv, en, no, da, fi, de, fr, es` (plus your configured
  default language). It uses the pipeline's requested language per call, otherwise the default.
- Multiple instances are supported — e.g. add a Swedish `kb-whisper-large` and a multilingual
  `whisper-1` side by side. The entity name includes the model so they are easy to tell apart.

Resulting entity: `stt.aurora_whisper_<model>` (the exact id is slugified by Home Assistant from
the instance name).

## Installation

1. Prerequisites
   - Home Assistant 2024.12 or later.
   - Access to `config/custom_components/` (File editor add-on, SSH, or Samba).
   - An API key for your OpenAI-compatible endpoint (e.g. `sk-st-...` for staik.se).

2. Copy the integration
   - Place the `custom_components/aurora_whisper/` folder into your HA `config/custom_components/`.
   - Or install via HACS (**HACS → Integrations → ⋮ → Custom repositories**, add
     `https://github.com/tonylofgren/aurora-whisper-stt`, category *Integration*).

3. Restart Home Assistant
   - **Settings → System → Restart**.

4. Add the integration
   - **Settings → Devices & services → Add integration**.
   - Search for **Aurora Whisper** and follow the config flow.

5. Verify
   - The integration appears under Devices & services and an `stt` entity is created.
   - Select the entity as the STT engine in a pipeline under **Settings → Voice assistants**.

## Configuration

Everything is configured through the UI (no YAML).

| Field | Description | Default |
|-------|-------------|---------|
| `base_url` | OpenAI-compatible API root | `https://api.staik.se/v1` |
| `api_key` | Bearer token (leave empty for servers without auth) | – |
| `model` | Model name, e.g. `kb-whisper-large` or `whisper-1` | `kb-whisper-large` |
| `language` | ISO-639-1 default language used when the pipeline doesn't request one | `sv` |
| `timeout` | Seconds to wait for a transcription | `30` |

**Options** — can be changed afterwards without removing the entity, via **Configure** on the
integration card: `model`, `language`, and an optional `prompt` (a context hint that improves
accuracy). Changes reload the entity automatically.

> **Note on validation:** during setup a best-effort check is made against `GET {base_url}/models`.
> Some servers answer `200` without validating the key, or lack `/models` entirely (then it is
> treated as "reachable"). A successful check therefore does not guarantee the key is valid — the
> only sure test is a real transcription.

## Troubleshooting

- **`invalid_auth` during setup** — the API key was rejected (HTTP 401/403). Check the key and
  that it belongs to the right `base_url`. For staik.se it should start with `sk-st-`.
- **`cannot_connect` during setup** — the `base_url` is wrong or the server is unreachable. Make
  sure the URL ends in `/v1` and test it from the same network as HA. Note that local TLS
  inspection (e.g. antivirus) can break verified HTTPS.
- **Empty or wrong transcription** — check that the model name exists on the server and the
  language is correct. Increase `timeout` if longer utterances get cut off. Set the log level
  `custom_components.aurora_whisper: debug` under `logger:` in `configuration.yaml` for details.

## Recovery

If the integration fails to load after a restart, open **Settings → System → Logs** (or
`config/home-assistant.log`) and search for `aurora_whisper`. Common causes are `ImportError`
(files copied incompletely — re-copy the whole folder) or `ConfigEntryError`/schema errors. If
the error persists, remove the integration entry under **Devices & services** and add it again;
no data is lost since the settings are just connection details.
